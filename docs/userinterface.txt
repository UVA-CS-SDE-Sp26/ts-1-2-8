Top Secret - Command Line Interface

Wesley Kleiman (A)

PURPOSE: This program is responsible for the command line interface used by the user to run the program.
         takes user commands and displays program results / errors

PSEUDOCODE:
- Instantiate ProgramController object in TopSecret class
- create logic method within TopSecret to make unit testing on TopSecret easier
- create set of if statements in logic to allow to condition output/operations on number of arguments given
-if:
 * 0 args given:
    - List numbered files to decipher/display
 * 1 arg given: Arg 1 -> file number
    - returns contents of file corresponding to number deciphered by default key
 * 2 args given: Arg 1 -> file number; Arg 2 -> deciphering key
    - returns contents of file deciphered by specified key

METHODS:

logic()
    - Arguments: String[] args
        -> sent directly from Main method arguments
    - Function:
        -> Use ProgramController to call appropriate logic (described in pseudocode) based on arguments given
    - Output:
        -> Print statements (Either list of files, deciphered file, or error)

    - Error handling:
        -> Check for invalid number of arguments (if-else using args.length)


DIRECT DEPENDENCIES:
- ProgramController - creates ProgramController (Part C) object and uses its methods to generate outputs

INDIRECT DEPENDENCIES:
- FileHandler - uses fileHandler (Part B) via ProgramController
- Cipher - deciphers file (Part D) via ProgramController


TESTING (Unit tests with Mock ProgramController)
 - Create mock ProgramController object to test start-to-end functionality
 - Configured to show functionality without using real file and logic in ProgramController

1) Check output format if zero arguments given
    - Expected output: check if listFiles() is called

2) Check output format if one argument given
    - Expected output: check if displayFile() is called and uses default key

3) Check output format if two arguments given
    - Expected output: check if displayFile() is called and uses specified key

4) Check to make sure error message shown if more than two arguments given
    - Expected output: Print error message
 


INTEGRATION TESTS:
Test using real ProgramController object to test complete compatability of program with various args.

1) Check invalid argument given (String / non-integer argument)
    - Expected output: Catch error in ProgramController.java

2) Check invalid argument given (too large - ex: 15)
       - Expected output: if statement recognize invalid argument in ProgramController.java

3) Check invalid second argument (not string)
    - Expected output: Catch error in fileHandler

4) No files available
    - Expected output: return error message from fileHandler section (Part B)

5) Check valid arguments (test both one and two)
    - Expected output: expected output for each context.
