Top Secret - Command Line Interface
2/10/26
Wesley Kleiman (A)

PURPOSE: This program is responsible for the command line interface used by the user to run the program.
         takes user commands and displays program results / errors

PSEUDOCODE:
- Instantiate ProgramController object in TopSecret class
- create logic method within TopSecret to make unit testing on TopSecret easier
- create set of if statements to allow to condition output/operations on number of arguments given
-if:
 * 0 args given:
    - List numbered files to decipher/display
 * 1 arg given: Arg 1 -> file number
    - returns contents of file corresponding to number deciphered by default key
 * 2 args given: Arg 1 -> file number; Arg 2 -> deciphering key
    - returns contents of file deciphered by specified key

ERROR HANDLING: Check for invalid number of arguments (if-else using args.length)

Dependencies:
- FileHandler - uses fileHandler (Part B) through ProgramController
- ProgramController - creates ProgramController (Part C) object and uses its methods to generate outputs


TESTING (Unit tests)
1) Check output format if zero arguments given
2) Check output format if one argument given
3) Check output format if two arguments given
4) Check to make sure error message shown if more than two arguments given
5) Check invalid argument given (not integer)
6) Check invalid second argument (not string)
- Create mock ProgramController object to test start-to-end functionality
 * Uses listFiles Method when 0 args given
 * Uses displayFile when 1 or 2 args given


INTEGRATION:
Test using real ProgramController object to test complete compatability of program with various args.
